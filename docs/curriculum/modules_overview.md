Lesson 1: What Is a Process? 
Objective: 
Understand that a process is a structured sequence of events: input -> processing -> output. 
Key Concepts: 
- A process always has a beginning, a transformation, and an end. 
 
- All processes follow logic, even human ones. 
 
- Machines and bodies follow similar process rules. 
 
Teacher Notes: 
Anchor all explanations in daily life. Processes aren't just digital; they're physical, emotional, mechanical. 
Analogy Examples: 
- Human: Hear a sound -> recognize danger -> run 
 
- Mechanical: Push elevator button -> logic checks floor -> elevator moves 
 
- Environmental: Sun rises -> temperature increases -> plants open 
 
Activity: 
Describe 3 familiar systems using this format: 
- Input: 
 
- Process: 
 
- Output: 
 
Checkpoint: 
Students write structured steps for brushing teeth and label which step is input, process, or output. 
Lesson 2: From Signals to Meaning 
Objective: 
Recognize how raw data is sensed, organized, and turned into information. 
Key Concepts: 
- Signals are sensed (light, sound, touch). 
 
- Data is raw, unorganized input. 
 
- Information is data that has been processed to have meaning. 
 
Teacher Notes: 
Use sensory metaphors: your ears pick up vibration, your brain turns it into music or language. 
Real-world Examples: 
- Thermometer senses temperature (data) -> digital display (information) 
 
- Student test scores (data) -> class average (information) 
 
Activity: 
Take a list of 10 numbers (random test scores). Ask: What's the highest score? What's the average? Now ask: what does this mean? 
Checkpoint:

## Page 8

Students label raw vs processed data in 3 examples. Must show understanding of the transition. 
Lesson 3: Structuring Storage 
Objective: 
Understand that how data is organized (list, stack, table) affects how it's used. 
Key Concepts: 
- Lists, stacks, and tables are data structures. 
 
- Structure changes how easily we access or process things. 
 
- Real-life systems rely on structured storage. 
 
Teacher Notes: 
Use examples students already know: backpacks, libraries, bookshelves, grocery shelves. 
Analogies: 
- Stack: pile of books (last in, first out) 
 
- Table: class seating chart 
 
- List: shopping list 
 
Activity: 
Students are given 6 items (e.g., apple, book, pen, shirt...). They must place them into three different structures and explain the difference. 
Checkpoint: 
Given a task (e.g. "find the oldest student"), students choose the best structure and justify it. 
Lesson 4: Instructions, Tasks, and Steps 
Objective: 
Learn that clear, ordered instructions form the basis of all systems and programs. 
Key Concepts: 
- Tasks are completed through steps. 
 
- The sequence matters. 
 
- Even small changes in order can break a system. 
 
Teacher Notes: 
Use physical demonstrations if possible. "Make a sandwich" is only simple if the steps are right. 
Activity: 
Write 5 steps for "getting dressed" or "making cereal." Then, randomize the order and identify what goes wrong. 
Checkpoint: 
Students must rewrite a scrambled process correctly. Must explain why their sequence works. 
Lesson 5: Decisions and Conditions 
Objective: 
Introduce conditional logic: systems respond differently based on inputs or states. 
Key Concepts:

## Page 9

- Conditions are decisions: "IF this, THEN that" 
 
- Systems don't "think," they follow rules 
 
- Conditions are everywhere (traffic lights, phone apps, teachers) 
 
Teacher Notes: 
Use branching path diagrams to reinforce: one input leads to two possible paths depending on a condition. 
Real-world Examples: 
- IF it's raining -> take umbrella 
 
- IF homework is finished -> go play 
 
Activity: 
Write 3 IF/THEN conditions based on school or home life. 
Checkpoint: 
Students must take a process from Lesson 4 and add a decision that changes the steps. 
Lesson 6: Repetition and Loops 
Objective: 
Understand that some steps repeat until a condition is met. 
Key Concepts: 
- Loops are repeated instructions. 
 
- They either run a fixed number of times, or until something changes. 
 
- Every loop needs an exit condition. 
 
Teacher Notes: 
Use rhythm and habit metaphors: brushing each tooth, climbing stairs, tying shoes. 
Real-world Examples: 
- "Wash until clean" 
 
- "Count to 10" 
 
- "Walk until the bell rings" 
 
Activity: 
Take a real-world activity (e.g. packing a bag). Identify the loop, define the ending condition. 
Checkpoint: 
Students must write one REPEAT UNTIL and one REPEAT N TIMES example using natural structured language.

Lesson 1: Turn a Process Into Code 
Goal: 
Transition from structured natural language to line-by-line step logic. Understand that a clear process can be written as a sequence of 
unambiguous instructions. 
Main Points: 
- Every action has an order. 
 
- Computers can only follow exact instructions. 
 
- Ambiguity in instruction breaks logic. 
 
- Instructions must show what to do, when, and how. 
 
Teacher Notes: 
Begin with messy everyday descriptions like "make a sandwich" or "clean the room." Show how unstructured language hides logic. Emphasize 
the mindset shift: the computer is not smart - it only does what it's told. 
Examples: 
Natural: 
"Brush your teeth before sleeping." 
Jargon: 
SET toothbrush ("dirty") 
SET toothpaste ("ready") 
SET mouth ("not clean") 
 
SET toothbrush ("clean") 
SET mouth ("clean") 
PRINT "You can go to sleep" 
Activity: 
Have students pick one daily routine and write it as 5-7 Jargon steps. 
Checkpoint: 
- Task must include at least one SET and one PRINT. 
 
- Must include a sequential process with no missing logic. 
 
- Score: 3/4 steps correct = pass. 
Lesson 2: Conditions and Comparisons 
Goal: 
Introduce conditional logic using comparisons. Show how IF statements control logic paths. 
Main Points: 
- Use comparisons: equal to, greater than, less than. 
 
- IF statements create forks in logic. 
 
- Each condition must evaluate to true or false. 
 
Teacher Notes: 
Use real-life examples: "IF it's raining, bring umbrella." Explain that every comparison evaluates a condition, not just checks.

## Page 15

Examples: 
SET temperature (15) 
IF temperature is less than 20 THEN 
 PRINT "Wear a jacket" 
END 
 
SET age (18) 
IF age is greater than or equal to 18 THEN 
 PRINT "You can vote" 
END 
Activity (Error-Based): 
Show broken logic: 
IF score > 90 THEN PRINT "Excellent" 
Students must rewrite in Jargon. 
Checkpoint: 
- One correct IF statement with proper structure. 
 
- One real-world logic path using a comparison. 
 
- Score: 2/2 needed to proceed. 
Lesson 3: Do the Math 
Goal: 
Learn to use arithmetic operations to update variables and make decisions. 
Main Points: 
- Arithmetic operators: +, -, *, /, % 
 
- SET can assign values or update them. 
 
- Math is used inside IF logic and loops. 
 
Teacher Notes: 
Focus on clarity and traceability. Use examples with scoring, prices, or basic finance. 
Examples: 
SET score (70) 
SET bonus (10) 
SET total (score + bonus) 
PRINT total 
SET price (150) 
IF price is greater than 100 THEN 
 SET price (price - 20) 
END 
PRINT price 
Activity:

Checkpoint: 
- At least one correct math operation. 
 
- Proper use of SET and IF with math. 
 
- Score: 2/3 operations work = pass. 
Lesson 4: Variables Remember Things 
Goal: 
Learn how variables hold, update, and pass information through a program. 
Main Points: 
- Variables store a value with a name. 
 
- Values can change; names stay the same. 
 
- Variables must be used consistently. 
 
Teacher Notes: 
Introduce mental models: think of variables as labeled boxes. Emphasize clarity of naming. 
Examples: 
SET energy (10) 
SET energy (energy - 2) 
PRINT energy 
SET name ("Leila") 
PRINT ("Hello, " + name) 
Activity (Fix-It): 
SET x x + 1 
Student must correct missing parentheses. 
Checkpoint: 
- Create and update two variables. 
 
- Print their final values correctly. 
 
- Score: 3/4 components correct = pass. 
Lesson 5: Lists and Looping Over Them 
Goal: 
Introduce lists and the concept of iterating over multiple items. 
Main Points: 
- Lists are collections of values. 
 
- Loops process each item one by one. 
 
- REPEAT_FOR_EACH simplifies indexed loops. 
 
Teacher Notes: 
Use a grocery bag, packing list, or guest list as a metaphor. Show how computers process one item at a time. 
Examples:

## Page 17

REPEAT_FOR_EACH name in guests 
 PRINT ("Hello, " + name) 
END 
With index: 
SET groceries (["milk", "eggs", "bread"]) 
SET i (0) 
REPEAT_UNTIL i reaches end of groceries 
 PRINT groceries[i] 
 SET i (i + 1) 
END 
Activity (Creative): 
Create a list of your top 3 favorite movies and print a custom message for each. 
Checkpoint: 
- Must create and iterate a list. 
 
- Output for each list item must be personalized. 
 
- Score: all items processed correctly = pass. 
Lesson 6: Paths and Possibilities 
Goal: 
Introduce branching logic using multiple IFs, including compound conditions. 
Main Points: 
- Use IF and ELSE for layered logic. 
 
- AND / OR combine conditions. 
 
- Only one branch runs in nested logic. 
 
Teacher Notes: 
Compare to flowcharts. Show how a program "chooses" its path. 
Examples: 
SET score (70) 
IF score is greater than or equal to 90 THEN 
 PRINT "Excellent" 
ELSE 
 IF score is greater than or equal to 70 THEN 
 PRINT "Good" 
 ELSE 
 PRINT "Try again" 
 END 
END

Compound condition:

SET humid (80)

 PRINT "Very hot and humid" 
END

Write a mood detector: 
- Happy if mood is "good" 
 
- Tired if energy is less than 3

- Must use at least one AND or OR. 
 
- Use ELSE or ELSE IF correctly.

Teacher Notes: 
Use examples like walking up stairs, drinking water until full, counting down a timer.

REPEAT_UNTIL i is equal to 5 
 PRINT i 
 SET i (i + 1) 
END

 PRINT i 
END 
Fix missing update step.

Module 3: Prompting 
Goal: 
Train students to communicate clearly and effectively with intelligent systems using natural language. Students will learn to write, evaluate, and 
refine prompts to achieve desired outcomes from text-based AI assistants.

## Page 23

Lesson 1: What Is a Prompt? 
Objective: 
Understand what a prompt is, why clarity matters, and how prompts differ from everyday questions. 
Key Concepts: 
- A prompt is an instruction given to an AI. 
 
- AI doesn't "know" what you want unless you tell it. 
 
- Specificity leads to better results. 
 
Examples: 
Bad: 
"Tell me about volcanoes" 
Good: 
"List 3 interesting facts about volcanoes for a grade 5 science project" 
Activity: 
Students are given 3 vague prompts. Their task is to rewrite each with a clear goal, context, and audience. 
Checkpoint Conversation: 
AI mentor replies with answers to both original and revised prompts to show differences. 
Lesson 2: Purpose First 
Objective: 
Learn to write prompts by starting with the desired output in mind. 
Key Concepts: 
- Think: What do I want the AI to give me? 
 
- Choose tone, length, format, and purpose 
 
- Make instructions match the goal 
 
Examples: 
Prompt goal: write a 3-line poem about rain 
Bad: 
"What is rain like?" 
Good: 
"Write a 3-line poem about rain using vivid sensory language" 
Activity: 
Students receive 4 goals (e.g. summarize, explain, write, compare) and must write purpose-first prompts for each. 
Checkpoint Conversation: 
The mentor executes their prompts and provides side-by-side feedback on whether the response matches the intended format and purpose. 
Lesson 3: Precision with Constraints 
Objective: 
Learn how to guide AI output using constraints like word count, audience, tone, or format. 
Key Concepts:

## Page 24

- Use phrases like "in 2 bullet points," "for a beginner," or "in formal tone" 
 
- Constraints help narrow the range of possible outputs 
 
- Without constraints, AI might ramble or miss your need 
 
Examples: 
Bad: 
"Explain DNA" 
Good: 
"Explain DNA in 2 short bullet points for an 8-year-old" 
Activity: 
Rewrite 3 prompts by adding at least 2 constraints each. 
Checkpoint Conversation: 
Mentor shows how small constraint changes (tone, length, audience) lead to big changes in the response. 
Lesson 4: Bad Prompt? Fix It. 
Objective: 
Learn how to recognize and improve poorly performing prompts. 
Key Concepts: 
- If AI gives irrelevant, vague, or overly long answers, the prompt was likely unclear. 
 
- Revising = key skill 
 
- Don't just accept - iterate 
 
Examples: 
Prompt: 
"Tell me about history" 
Output: 
AI gives a vague essay. 
Fix: 
"Give a one-paragraph summary of World War II for middle school students" 
Activity: 
Students test 2 bad prompts with the mentor, then fix and retry them. 
Checkpoint Conversation: 
Mentor reflects on the revised prompt quality and encourages a second improvement. 
Lesson 5: Multi-Step Prompts 
Objective: 
Learn how to break down tasks into multiple instructions within one prompt. 
Key Concepts: 
- AI follows sequential instructions 
 
- You can ask for step-by-step outputs

## Page 25

- Structure matters: list, explain, summarize, etc. 
 
Examples: 
Prompt: 
"Explain how to build a kite and list 3 common mistakes" 
Expected output: 
Step-by-step build + mistake list 
Activity: 
Write 2 prompts that include: 
1. A task 
 
2. A follow-up condition (e.g. "Then...") 
 
Checkpoint Conversation: 
Mentor checks whether both parts of the prompt were followed correctly, and if not, asks the student to refine structure. 
Lesson 6: Comparative and Analytical Prompts 
Objective: 
Use prompting to compare, evaluate, or analyze rather than just receive facts. 
Key Concepts: 
- Higher-order prompts = deeper learning 
 
- Good for essays, arguments, or summaries 
 
- Ask AI to "Compare A and B," "Evaluate...," or "Decide which is better and why" 
 
Examples: 
Bad: 
"What is a cat?" 
Good: 
"Compare cats and dogs as pets in terms of behavior, space, and care needs" 
Activity: 
Write 2 comparative prompts. Test one live with mentor. 
Checkpoint Conversation: 
Mentor highlights depth and logic. If vague, prompts must be made more precise with categories or criteria. 
Lesson 7: Conversation and Follow-Up 
Objective: 
Learn how to carry conversations with AI assistants using follow-ups and refinement. 
Key Concepts: 
- One prompt isn't always enough 
 
- You can ask the AI to "explain more," "give an example," or "make it shorter" 
 
- Dialogue is an iterative process 
 
Example Flow:

## Page 26

1. Prompt: 
 
"Write a paragraph on climate change" 
2. Follow-up: 
 
"Now turn that into 3 bullet points" 
3. Refine: 
 
"Make it suitable for 10-year-olds" 
Activity: 
Students must complete a 3-turn conversation with the mentor, refining a prompt twice based on feedback. 
Checkpoint Conversation: 
Mentor evaluates the full prompt-response-refinement loop for clarity and effectiveness. 
Lesson 8: Final Prompting Challenge 
Objective: 
Demonstrate mastery of prompt structure, clarity, and adaptability. 
Challenge Task: 
Create three prompts of different types: 
1. Creative (e.g. write a story) 
 
2. Informational (e.g. explain a process) 
 
3. Comparative/analytical (e.g. compare two things) 
 
Each must: 
- State a clear purpose 
 
- Use at least one constraint 
 
- Be tested with the AI mentor 
 
- Be refined at least once if needed 
 
Checkpoint Conversation: 
Mentor evaluates each of the three outputs. Student must pass all three to complete the module.

BREAK BREAK Immediately exits the nearest loop
